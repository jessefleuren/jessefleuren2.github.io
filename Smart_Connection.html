Smart Connection

START OVER MIJ SMARTIE PROJECT

Smart Connection. De titel van het vak geeft nog niet zoveel weg. Maaaaarrr... je kijkt toch naar het resultaat. Smart Connection is het middel van de minor om web development aan te leren aan de studenten. Het ontwikkelen van deze website (of portfolio) is ook onderdeel van de beoordeling in de vorm van een portfolio.

Web devolopment is voor mij helemaal nieuw, ik heb er nog niet eerder mee gewerkt en het leek bij aanvang ook erg ingewikkeld. Niets is minder waar. Zeker, het is even wennen, omdat sommige principes tegen je eigen intuïtie ingaan, maar zodra ik een beetje de basisprincipes doorhad, ging het eigenlijk wel. Om een beeld te geven in hoe deze website tot stand gekomen is, geef ik graag een inkijkje in mijn denkproces bij het ontwerpen van een eigen website. Ook laat ik graag zien welke bronnen ik gebruikt heb, want alleen had ik het nooit gekund! Zie deze pagina dan ook als een soort verantwoording voor wat wel en wat niet gelukt is tijdens het ontwikkelen van dit portfolio.

Week 1 t/m 4: HTML/CSS/JS

Week 1, we zijn begonnen aan de minor Smart Industry. De introdag was een beetje rommelig, maar het belangrijkste voor mij was dat ik wist hoe de minor in grote lijnen ging lopen, maar bovenal wat er van mij verwacht ging worden. Nog voor de minor begon werd duidelijk dat we een project gingen doen bij een bedrijf. Dat wist ik al, en lag ook binnen mijn comfort zone. Echter, aan het einde zou ik ook als individu een opdracht moeten doen; het maken van een portfolio. Ook nog: binnen de comfort zone. Alleen was het niet zoals ik al jaren portfolio's in elkaar heb gesleuteld. Nu was de opdracht om mijn leerproces te toetsen aan competenties en te laten zien dat je ze beheerst op een echte, eigen, webpagina! Heftig!

Op de eerste dag worden we gelijk aan het knippen en plakken gezet, maak maar een mock-up van hoe jij nu denkt dat je portfolio eruit moet gaan zien, ongeacht over wat je kan of niet kan in het brede scala mogelijkheden van web development.

Stapje voor stapje worden we tijdens de lesweken meegenomen in de mogelijkheden en toepassingen van web development. Dit betekent eerst begrijpen wat HMTL inhoud, en hoe ik code kan schrijven. Dit doe ik door een aantal opdrachten te maken, en een klein stukje portfolio te beginnen. Later komt de opmaak daarbij kijken, en de opmaak wordt vormgegeven door CSS. Ook hier zijn opdrachten vanuit de minor beschikbaar, maar heb ik bewust gekozen om juist de opgedane kennis gelijk toe te passen op mijn website. Met HTML en CSS is al veel mogelijk en ga ik ook een aantal weken 'los'. Waarbij andere mensen losgaan zien als het creëren van duizenden pagina's, resulteert het bij mij juist in het perfectioneren van één pagina. Al snel merk ik dat ik gewoon niet de tijd heb om een perfecte en supergoede webdeveloper te zijn. Mijn perfectionisme moet ik opzij zetten, en genoegen nemen met wat ik kan en op tijd af kan krijgen. Ik wist echter al wel snel hoe ik mijn pagina vorm wilde geven; navigatiebalk onderin, om bovenin juist ruimte vrij te houden. De intropagina is niet scrollbaar, maar dient als soort van voorpagina waarop een grote foto van mij te zien is. Verder wilde ik graag vasthouden aan het concept van de eerste dag; de plaknotities. Daarom heb ik met behulp van w3schools geleerd hoe ik een soort polaroid shot uitprintsel kon maken, om het geheel er toch uit te laten zien alsof ik met plaknotities heb gewerkt. Dat was de achterliggende gedachte.

Ook hebben we geleer hoe JavaScript werkt, dit is in de les gedaan door kleinere opdrachten, maar wilde ik buiten de les ook graag liever weer toepassen op mijn eigen website dan een andere opdracht. Daarom vind ik mijn startpagina nu ook zo goed gelukt. Ik heb hier namelijk zelf (!) een 'random quote generator' gebouwd zoals je wellicht al hebt kunnen zien. Hier zijn echt talloze uren in gekropen maar mijn perfectionisme wilde ten minste dat dat lukte. Keer op keer heb ik; gefaald, een errormelding, geen melding maar ook geen tekst zichtbaar, een knop die niet werkt, verkeerde notatie in javascript, en ga zo maar door. Maar uiteindelijk was het daar; precies zoals ik het wilde; een knopje met telkens een nieuw stukje tekst dat gehaald wordt uit een kleine database.

Week 5: Weersvoorspelling maken

Met behulp van een stukje theorieles over JSON, leerde ik in de praktijk wederom een opdracht maken. Echter, dacht ik dat het nu leuk zou zijn om dit te integreren in mijn portfolio.
Kijk gerust even wat het weer vandaag doet in Nijmegen! Tijdens de opdracht heb ik met behulp van een eigen code geleerd hoe ik data vanuit een server op kan vragen. De technologie erachter is iets dat bij mij minder blijft plakken als ik het oplees, maar het mooie van pielen met HTML en dergelijke is; 'trial and error'. Ik kan zelf bepalen welke gegevens ik opvraag door ze te integreren bovenin bij divID en onderin door GetElementByID bij het kopje Function. Hierdoor leer ik dat ik ook andere zaken kan opvragen; die bij het inspecteren van de HTML pagina kan vinden. Nu is het een kwestie van invoegen uit en uitvoeren.

Week 6: Theorieles over machinelearning

In de laatste week van Smart Connection is het advies vooral; werk vanaf nu aan je portfolio. In deze afsluitende week kreeg ik nog wel te weten hoe 'machinelearning' nou precies in zijn werk gaat. Ik heb hier tijdens mijn studie vaak over gehoord maar wist niet exact wat het inhield en hoe het in zijn werk gaat. Het aanleren van een actie aan machines gaat nog niet zo makkelijk, nog maar eens een reminder over hoe speciaal wij als mensen zijn eigenlijk.

Bij deze les heb ik een model gemaakt waarin ik heb getracht de juiste connecties te maken tussen de stappen die volgen vanuit een dataset om tot een applicatie te leiden. Dagelijks gebruik ik tientallen apps, maar hoe deze aan hun gegevens komen is vaak nog onduidelijk, en hoe iets leid tot de 'front-end', op mijn scherm. Hieronder zie je hoe ik denk dat de stappen verlopen.
